## Tools

### Tool 1: search_listings

**What it does:**
Searches the mock listings dataset (`data/listings.json`) for items that match the user's description, size, and price constraints.

**Input parameters:**
- `description` (str): Keywords describing what the user is looking for (e.g., "vintage graphic tee").
- `size` (str | None): Size string to filter by. Matching should be case-insensitive (e.g., "M").
- `max_price` (float | None): Maximum price boundary (inclusive).

**What it returns:**
A list of dictionaries (`list[dict]`) containing the matching listing items, sorted by relevance (keyword overlap score).

**What happens if it fails or returns nothing:**
If no listings match the criteria, it returns an empty list `[]`. It does *not* raise an exception. The agent planning loop will catch this empty list and alert the user to adjust their search.

---

### Tool 2: suggest_outfit

**What it does:**
Acts as a virtual stylist by taking a newly found thrift item and suggesting 1–2 complete outfits incorporating pieces from the user's existing wardrobe dataset.

**Input parameters:**
- `new_item` (dict): The listing dictionary of the item the user is considering buying.
- `wardrobe` (dict): A wardrobe dictionary containing an 'items' list.

**What it returns:**
A string (`str`) containing a paragraph of specific styling advice and outfit combinations generated by the LLM.

**What happens if it fails or returns nothing:**
If the user's wardrobe list is empty, it gracefully prompts the LLM to offer *general* styling advice for the item rather than crashing or returning an empty string.

---

### Tool 3: create_fit_card

**What it does:**
Generates a short, shareable outfit caption for social media (Instagram/TikTok) based on the outfit suggestion and the specific item details.

**Input parameters:**
- `outfit` (str): The outfit suggestion string generated by `suggest_outfit()`.
- `new_item` (dict): The listing dictionary for the thrifted item.

**What it returns:**
A 2–4 sentence string (`str`) that acts as a casual, authentic caption mentioning the item name, price, platform, and vibe.

**What happens if it fails or returns nothing:**
If the `outfit` input is empty or missing, it returns a descriptive error message string rather than crashing the agent.

## Planning Loop

**How does your agent decide which tool to call next?**
The agent operates using a sequential, state-dependent loop inside `run_agent()`. 
1. It parses the user query to extract search arguments.
2. It calls `search_listings`. *Condition:* It checks if the returned list is empty. If it is empty, it sets the `session["error"]` message and halts immediately.
3. If listings are found, it extracts the top result, sets it in state, and calls `suggest_outfit`.
4. Finally, it passes the resulting outfit and item to `create_fit_card`.

---

## State Management

**How does information from one tool get passed to the next?**
State is tracked within a single `session` dictionary initialized by `_new_session()` at the start of the interaction. When a tool finishes executing, its output is saved into specific keys (e.g., `session["search_results"]`, `session["selected_item"]`, `session["outfit_suggestion"]`). The next tool in the sequence reads its inputs directly from this session dictionary without needing to ask the user for duplicate information.

---

## Error Handling

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| `search_listings` | No results match the query | Sets `session["error"]` with a message asking the user to adjust constraints and halts the agent. |
| `suggest_outfit` | Wardrobe is empty | Detects the empty list and instructs the LLM to provide general styling advice instead of referencing specific owned items. |
| `create_fit_card` | Outfit input is missing or incomplete | Returns a safe fallback error string instead of crashing the pipeline. |

## A Complete Interaction (Step by Step)

FitFindr is an AI agent that orchestrates a multi-step thrifting workflow. It first uses `search_listings` to find items matching a user's constraints, passes any found item to `suggest_outfit` to style it with their existing wardrobe, and finally calls `create_fit_card` to generate a shareable caption. If a tool fails (like `search_listings` returning zero results), the agent halts the sequence and alerts the user to adjust their search instead of passing empty data forward.

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The agent calls `search_listings(description="vintage graphic tee", size=None, max_price=30.0)` to find matching items from the dataset.

**Step 2:**
The agent takes the top returned listing and calls `suggest_outfit(new_item=<top_listing_dict>, wardrobe=<user_wardrobe_dict>)` to generate styling advice using the baggy jeans and chunky sneakers.

**Step 3:**
The agent calls `create_fit_card(outfit=<outfit_string>, new_item=<top_listing_dict>)` to generate a short social media caption about the thrifted find.

**Final output to user:**
The user sees the top listing details, a paragraph of personalized styling advice incorporating their wardrobe, and a short, shareable fit card caption.
